# Library Management System

Информационная система для управления библиотекой, выполненная в рамках учебного задания.

## Технологии
- **Java 17**
- **Spring Boot 4.x** (Data JPA, Web, Security, Validation)
- **PostgreSQL** (Основное хранилище: книги, читатели, выдачи)
- **ClickHouse** (Аналитическое хранилище: статистика выдач)
- **Liquibase** (Миграции базы данных)
- **Lombok & MapStruct** (Уменьшение шаблонного кода и маппинг DTO)

## Функциональные возможности
### Работа с книгами
- Добавление новой книги
- Просмотр списка всех книг
- Поиск книги по названию

### Работа с читателями
- Регистрация новых читателей
- Просмотр списка зарегистрированных читателей

### Операции выдачи
- Выдача книги читателю (с проверкой наличия экземпляров)
- Возврат книги
- Просмотр активных выдач конкретного читателя

### Статистика (ClickHouse)
- Показ популярных книг (на основе количества выдач)
- История выдач по читателю

## Архитектура
Проект построен по классической многослойной архитектуре с разделением ответственности (SOLID):
- **Controller**: Обработка HTTP-запросов.
- **Service**: Бизнес-логика приложения.
- **Repository**: Взаимодействие с базами данных (JPA для PostgreSQL и JdbcTemplate для ClickHouse).
- **DTO**: Объекты передачи данных для API.
- **Mapper**: Преобразование между сущностями и DTO.

## Настройка и запуск

### Требования
- JDK 17
- Docker & Docker Compose (рекомендуется)

### Переменные окружения
Приложение использует переменные окружения для настройки подключений. Вы можете создать файл `.env` в корне проекта или настроить их в IDE:
- `DB_URL`: URL к PostgreSQL (например, `jdbc:postgresql://localhost:5432/library`)
- `DB_USER`: Пользователь PostgreSQL
- `DB_PASS`: Пароль PostgreSQL
- `CLICKHOUSE_URL`: URL к ClickHouse (например, `jdbc:clickhouse://localhost:8123/default`)
- `CLICKHOUSE_USER`: Пользователь ClickHouse
- `CLICKHOUSE_PASS`: Пароль ClickHouse
- `JWT_SECRET`: Секретный ключ для JWT (если используется безопасность)

### Схема ClickHouse
Для работы аналитики необходимо создать таблицу в ClickHouse:
```sql
CREATE TABLE IF NOT EXISTS loan_stats (
    book_id UInt64,
    reader_id UInt64,
    issued_at DateTime
) ENGINE = MergeTree()
ORDER BY (issued_at, book_id);
```

### Запуск
1. Склонируйте репозиторий.
2. Настройте базы данных (PostgreSQL и ClickHouse).
3. Запустите приложение через Gradle:
   ```bash
   ./gradlew bootRun
   ```
